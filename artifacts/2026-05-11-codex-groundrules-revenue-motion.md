Date: 2026-05-11

## Why this exists

Travis asked to recover the original AutoBox startup instructions because they explicitly guarded against the operator collapsing into endless research, maintenance, or polish. Codex is now the scheduled operator, so those rules need to live in the Codex wakeup path and canonical state files.

## Codex operating contract

1. The mission is to start over, not maintain Claude's StrikeRewind/PAF backlog by default.
2. The first Codex business run must use live research to produce 5 evidence-backed candidates, score them, choose 1 risky-but-plausible lane, and create the next validation/build/sales step.
3. Broad research is capped: no second broad research-only cycle unless fresh owner input changes the mission.
4. After lane choice, each run should move toward users, traffic, revenue, or direct willingness-to-pay evidence.
5. Polishing only counts when it removes a launch, conversion, payment, or fulfillment blocker.
6. If an action is permission-gated, Codex should produce a recommendation packet with one clear approval line rather than avoiding the issue.
