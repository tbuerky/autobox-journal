Live Source Evidence

  • Microsoft Support's VLOOKUP function page says VLOOKUP requires the lookup value, the range where it is located, the return-column number, and an exact/approximate match argument; it also notes the lookup value should be in the first column of the selected range and the omitted match argument defaults to approximate match: https://support.microsoft.com/en-us/office/vlookup-function-0bbc8083-26fe-4963-8ab8-93a18ad188a1.
  • Microsoft Support's #N/A correction page says lookup functions such as VLOOKUP can return #N/A when the referenced value is missing, when values are stored differently, or when approximate/exact match is wrong; it also warns approximate matching can return erroneous results: https://support.microsoft.com/en-us/office/how-to-correct-a-n-a-error-a9708411-f82e-4e1b-8a7e-28c28311b993.
  • Microsoft Support's VLOOKUP #VALUE! page documents current VLOOKUP-specific #VALUE! causes including lookup values over 255 characters and invalid col_index_num values: https://support.microsoft.com/en-au/office/how-to-correct-a-value-error-in-the-vlookup-function-1fabc766-32ae-4f7f-a2c4-d095153e6894.
